## Runtime Dependencies

`archive_marketing.py` uses **only the Python standard library**. There are no `pip install` requirements.

| Module | Used for |
|--------|---------|
| `argparse` | CLI flag parsing and help generation |
| `csv` | CSV export of classification decisions |
| `datetime` | Timestamp generation, date range computation, email date parsing |
| `json` | Config file parsing, MCP request/response serialization |
| `os` | Environment variable access, file permission checks |
| `pathlib.Path` | Cross-platform path construction for default config/connection paths |
| `re` | Compiled regex patterns for classification |
| `stat` | POSIX permission bit constants for connection file security check |
| `sys` | `sys.exit()` for fatal errors, stdout/stderr reconfiguration |
| `time` | `time.sleep()` for rate limiting, `time.time()` for run duration |
| `urllib.request` | HTTP POST requests to the MCP bridge |
| `urllib.error` | `URLError` type for network error handling |

**Minimum Python version:** 3.8

The 3.8 minimum is set by:
- `sys.stdout.reconfigure()` — available since 3.7
- `f-strings` — available since 3.6
- `typing` with `Dict`, `List`, `Optional`, `Tuple` — available since 3.5
- The actual limiting factor is CI test coverage, which starts at 3.8

## Development Dependencies

Used only in CI — not required to run the script:

| Tool | Version | Purpose |
|------|---------|---------|
| `ruff` | latest | Linting (`ruff check archive_marketing.py tests/`) |
| `pytest` | optional | Alternative test runner (script works with stdlib `unittest`) |

Install for local development:

```bash
pip install ruff
# pytest is optional — unittest works without it:
python -m unittest tests.test_classification -v
```

## External System Dependencies

These are not Python packages but required system components:

### Mozilla Thunderbird

| Attribute | Details |
|-----------|---------|
| **What it is** | Desktop email client (IMAP/POP3/SMTP) |
| **Why required** | Acts as the IMAP client; the script delegates all credential management and IMAP protocol handling to it |
| **Version** | Any recent version (tested with Thunderbird 115+) |
| **License** | Mozilla Public License 2.0 |
| **Must be running** | Yes — the thunderbird-mcp extension only runs when Thunderbird is open |

### thunderbird-mcp Extension

| Attribute | Details |
|-----------|---------|
| **What it is** | Thunderbird extension that exposes a local JSON-RPC HTTP server |
| **Repository** | [github.com/joelpurra/thunderbird-mcp](https://github.com/joelpurra/thunderbird-mcp) |
| **Why required** | Provides the API through which the script issues IMAP read and move commands |
| **Protocol** | MCP (Model Context Protocol) JSON-RPC 2.0 over HTTP, localhost only |
| **Auth** | Bearer token, rotated each Thunderbird restart |
| **License** | See repository |

## Security Considerations

### No credential storage

The script never stores, logs, or transmits email credentials. Thunderbird handles all IMAP authentication internally.

### Bearer token lifetime

The thunderbird-mcp bearer token is a local session secret with the following properties:
- Generated by Thunderbird on extension start
- Stored in a temp file (`connection.json`) that should be `chmod 600` on POSIX
- Valid only while Thunderbird is running
- Invalidated when Thunderbird restarts

The script warns if `connection.json` has permissive permissions on POSIX systems.

### Network isolation

All HTTP communication is to `http://127.0.0.1:{port}/` — strictly localhost. The script makes no outbound network connections of any kind.

### No email body access

Only `sender` (author) and `subject` fields are read from the MCP response. Email bodies are never requested or processed.

### Supply chain risk

Since the script has no `pip` dependencies, there is no supply chain attack surface from PyPI packages. The only external code execution is Python itself and the thunderbird-mcp extension (which runs inside Thunderbird, not in the script's process).

### Regex denial of service (ReDoS)

All regex patterns are compiled once at module load time. The patterns use `re.VERBOSE` with explicit alternation — no nested quantifiers or ambiguous patterns that could cause catastrophic backtracking. Email subjects and sender strings are short, bounded inputs, making ReDoS risk negligible.
